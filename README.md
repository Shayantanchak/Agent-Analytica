# 🚀 Analytica

> AI-Powered Multi-Agent Data Analyst for Automated Insights, Reports & Presentations

---

## 🧩 Overview
Analytica is an intelligent, multi-agent data analytics system that transforms raw datasets into decision-ready insights, structured reports, and executive PowerPoint presentations.

It automates the entire analytics pipeline:
**Data ingestion → Cleaning & profiling → KPI analysis → Visualization → Insight generation → Report & PPT creation**

---

## ⚡ Key Features
- 📊 Automated Data Profiling & Cleaning
- 🧠 Multi-Agent Analysis Engine
- 📈 KPI & Trend Detection
- 🚨 Anomaly Detection (IQR Bounds)
- 🧾 Auto Report Generation (DOCX)
- 📽️ PowerPoint Deck Generation (Premium Widescreen UI)
- ✅ QA Validation Layer (Hallucination Mitigation)
- 🔁 Audit Trail & Immutable Run History
- 📡 Google Sheets Integration Ready

---

## 🏗️ Architecture
User → Orchestrator → Data Agents → Analysis Agents → QA Validation → Output Engines (Report + PPT)

### Agents Implemented:
1. **Profiler Agent:** Computes Data Quality Scores and maps schemas.
2. **Cleaning Agent:** Normalizes columns, imputes missing data, caps outliers.
3. **Analysis Agent:** Generates Plotly charts, KPIs, and LLM Narratives.
4. **QA Agent:** Defends against unverified claims via strict numeric cross-referencing.
5. **PPT & Report Agents:** Transforms dictionary outputs into `.pptx` and `.docx` artifacts.

---

## 🧪 Tech Stack
- **Python Backend:** Pandas, NumPy, Scikit-learn
- **Frontend App:** Streamlit (Custom "AI Dashboard Pro" UI Theme)
- **Visualization:** Plotly, Kaleido
- **Generators:** python-pptx, python-docx

---

## 🚀 Getting Started
1. Clone the repository natively.
2. Install the necessary dependencies: `pip install -r requirements.txt` (Ensure `kaleido`, `python-pptx`, and `python-docx` are installed for the generators to work).
3. Start the UI:
   ```bash
   streamlit run app/ui_streamlit.py
   ```
4. Access the portal at `http://localhost:8501`.

# 🚀 Phase 1-5 Upgrade Blueprint & Refactor Plans

> **Note:** The cleanly upgraded pipeline implements these features as interconnected phases, integrating deeply with automation limits for Sheets, Excel, and PowerBI.

**The Upgraded Flow:**
Ingest → Profile/Clean → Analyze → Validate → Explain → Recommend → Export → Sync to Excel / G-Sheets / Power BI

### Target Phase 0 Project Structure (Services)
```text
project/
├── app/                  # UI and API routes
├── core/                 # Orchestrator, logs, schemas
├── modules/              # Profiling, cleaning, anomaly, forecast, etc.
├── outputs/              # PPT, Docs, Charts
├── connectors/           # Excel, G-Sheets, PowerBI
├── storage/              # DB runs, artifacts
└── tests/
```

### Phase 1 — QA Validation Layer (First Priority)
Protects the system from writing unsupported claims, inconsistent KPIs, or chart-text mismatches. Contains:
- `qa/metric_validator.py`
- `qa/chart_validator.py`
- `qa/narrative_validator.py`
- `qa/contradiction_checker.py`
- `qa/confidence_scorer.py`

### Phase 2 — Anomaly Detection Module
Rule-based (sudden drops, duplicate IDs) vs. ML-based (Isolation Forest, IQR bounds for numeric targets). Generates flags and UI charts.

### Phase 3 — Forecast Model
Includes time-series analysis (moving average, ARIMA). Contains logic for target detection, aggregation, training, selection, and explanation.

### Phase 4 — Explainable Insights Module
Attaches reasoning and evidence lineage to all conclusions generated by the system.

### Phase 5 — Recommendation Engine
Maps anomaly and forecast predictions to executable business actions ranked by impact, effort, and confidence.

### Detailed Integration Phases
* **Phase 6:** Orchestrate all together
* **Phase 7:** Excel Integration (Local + Microsoft Graph via output sheets)
* **Phase 8:** Google Sheets Integration (REST APIs via GC project)
* **Phase 9:** Power BI Integration (Clean datasets + PBI Dashboard mapping)

*This blueprint establishes the final foundational architecture for the Analytica Agent.*
